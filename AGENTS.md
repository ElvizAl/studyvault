# AGENTS.md

## Stack

TanStack Start (file-based routing) + Nitro server adapter + Prisma ORM (PostgreSQL) + Better Auth + Biome + Tailwind CSS v4 + shadcn/ui.

## Commands

```bash
pnpm dev          # Vite dev server on port 3000
pnpm build        # Vite build → .output/
pnpm test         # Vitest (no test files yet)
pnpm check        # Biome lint+format --write --unsafe (preferred one-liner)
pnpm lint         # Biome lint only
pnpm format       # Biome format only
```

After `pnpm install`, Prisma client auto-generates via `postinstall` script.

## Verification order

`pnpm check` → `pnpm build`. CI runs exactly this sequence on PRs to master.

## Architecture

```
src/
  routes/          # TanStack file-based routes (auto-generated routeTree.gen.ts)
  modules/         # Domain modules: auth, note, notebook, ai
  shared/          # Shared libs (prisma, auth-client, utils), middleware, components
  components/ui/   # shadcn/ui primitives
  generated/prisma # Prisma client (auto-generated, gitignored)
```

- **Path aliases**: `#/` and `@/` both resolve to `src/` (configured in tsconfig.json and package.json imports).
- **Server functions**: Use `createServerFn` from `@tanstack/react-start` with `.middleware([dbMiddleware])` for DB access. Defined in `modules/*/*.api.ts`.
- **Auth**: Better Auth with email+password and Google OAuth. Session checked via `requireSessionFn` or `auth.api.getSession()`.
- **DB middleware**: `src/shared/middleware/db.middleware.ts` — injects Prisma client into server function context.
- **AI summaries**: Uses OpenAI SDK (OpenRouter, deepseek model). Requires `OPENAI_API_KEY` env var. Notes need ≥150 words.

## Generated code — do not edit

- `src/routeTree.gen.ts` — auto-generated by TanStack Router plugin. VS Code marks it read-only.
- `src/generated/prisma` — Prisma client output. Regenerate with `npx prisma generate`.

## Linter config (Biome)

- Formatter: tabs, double quotes for JS/TS.
- Ignores: `routeTree.gen.ts`, `styles.css`.
- Some a11y and suspicious rules disabled (see `biome.json`).

## Deployment

- **Branch**: `master` is the deploy branch.
- **CI** (PRs): `npm ci` → `npx biome check --write --unsafe` → `npm run build`.
- **Deploy** (push to master): SSH into server, pull, `npm ci`, build, `prisma migrate deploy`, `pm2 restart studyvault`.
- **Runtime**: Node 24, PM2 process manager, output at `.output/server/index.mjs`.

## Environment variables

Required in `.env` or production env:
- `DATABASE_URL` — PostgreSQL connection string (Neon)
- `BETTER_AUTH_SECRET` — auth secret
- `BETTER_AUTH_URL` — base URL (default `http://localhost:3000`)
- `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` — Google OAuth
- `OPENAI_API_KEY` — for AI summaries (OpenRouter, deepseek model)

## Gotchas

- UI error messages are in Indonesian (locale: `id`).
- `pnpm` is the local package manager, but CI uses `npm ci` — keep scripts compatible with both.
- Nitro is pinned to nightly (`npm:nitro-nightly@latest`) — watch for breaking changes.
- Soft deletes used for notes and notebooks (`deletedAt` field). Always filter by `deletedAt: null` in queries.
